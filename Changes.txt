Changes for 0.1.4 (2016-11-26)
--------------------------------------------------------------------------------

New features:
* 検査結果が Unicode 以外の文字コードになる場合に文字コードを指定できるようにした
* RedHat7系の検査シナリオ追加。主な変更点はネットワークとサービス周りの検査
* 以下検査シナリオを見直し
    * Windows 検査シナリオ
        - [x] OS名、バージョン、サービスパック、アーキテクチャ(32bit or 64bit)情報
        - [x] 物理搭載メモリ量、仮想メモリ量と分ける
        - [x] ネットワーク サブネットマスク
        - [ ] NTPサーバIPアドレス "w32tm /query /status" コマンドで検索
    * Linux 検査シナリオ
        - [ ] ネットワーク サブネットマスク。クラス表記(x.x.x.x/24など)からサブネットマスクに変換
        - [ ] NTPサーバIPアドレス。"/usr/sbin/ntpq -p" コマンドで検索
        - [ ] IPv6ファイヤーウォールの有無。"chkconfig ip6tables --list" コマンドで検索
        - [ ] restorecond(SELinux の運用サポートデーモン)の有無
        - [ ] OSアカウント情報
        - [ ] NetworkManager(/etc/rc.d/init.d/NetworkManager?) の検索
    * VMWare ESXi ホスト検査シナリオ
        - [ ] vCenterアカウント情報
        - [ ] ネットワークデバイス情報の桁連れ問題
        - [ ] データストアデバイス情報の桁連れ問題
        - [ ] アフィニティルール
    * SC3000 ストレージ
        - [ ] ストレージ構成情報。 tsuacs コマンドで検索

Bug fixes:
*

Changes for 0.1.3 (2016-11-14)
--------------------------------------------------------------------------------

New features:
* ルール評価の評価式でserver_infoマップ変数を利用できるようにした

Bug fixes:
* ルール評価で値が数値の場合に型変換エラーになる問題対処
* config.groovy で test.dry_run_staging_dir パラメータが有効にならない問題対処
* 設定ファイルの読み込み時のエンコードエラーの対処
* getconfig.batで空白を含むディレクトリの下で実行した場合にjarパスが見つからない問題対処

Changes for 0.1.2 (2016-11-7)
--------------------------------------------------------------------------------

New features:
* vCenter 用 PowerShell スクリプトに例外処理追加、初回のログイン処理に失敗したら終了する
* Windows 用 PowerShell スクリプトに Enter-PSSession コマンドを追加して、セッションを保持する様に変更
* 検査結果がnull(空欄)の場合はエビデンスの結果セルをNGに配色する

Bug fixes:
* PowerShell の例外処理発生時にエラーが出力されない問題対処
* Linux 検査のSSHセッション例外発生時にプログラムが終了する問題対処

Changes for 0.1.1 (2016-10-31)
--------------------------------------------------------------------------------

New features:
* 検査ID絞り込みオプションを追加
  getconfig -s {サーバ名} -t {検査ID} とすることで指定した検査のみ実行
* 検査対象シートを横持から縦持ちに変更。設定項目を追加して検査シナリオで設定値を使用出来る様に変更
* test.serialization.tasksで並列化させないタスクを指定。
  vCenter情報採取などのケースで多重化した場合に検査対象側サーバに負荷が集中する場合に指定する
* Linux,Windows,vCenterの不具合検査項目を修正
* ドキュメント 使用方法、開発者ガイドを作成

Bug fixes:
* SSH ワークディレクトリ削除エラーの例外対処
* 非仮想化サーバでvCenter検査が実行されてしまう問題対処
